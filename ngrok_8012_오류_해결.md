# 🚨 ngrok ERR_NGROK_8012 오류 해결 가이드

## 🔍 오류 원인
- **오류 코드**: `ERR_NGROK_8012`
- **원인**: ngrok이 로컬 서버 `http://localhost:3000`에 연결할 수 없음
- **상태**: 서버가 실행되지 않음

## 🛠️ 해결 방법

### 1단계: 서버 상태 확인
```cmd
netstat -an | findstr ":3000"
```
- 결과가 나오면 서버 실행 중
- 결과가 없으면 서버가 실행되지 않음

### 2단계: 서버 시작
```cmd
# 방법 1: 자동 스크립트
서버_시작.bat

# 방법 2: 수동 실행
npm run dev
```

### 3단계: 서버 확인
브라우저에서 `http://localhost:3000` 접속 테스트
- 접속되면 서버 정상
- 접속 안되면 서버 문제

### 4단계: ngrok 터널 재시작
```cmd
# 기존 ngrok 터널 종료 (Ctrl+C)
# 새로 시작
ngrok http 3000
```

## 🎯 단계별 해결 순서

### 즉시 해결 방법
1. **새 CMD 창 열기**
2. **서버 시작**: `npm run dev`
3. **서버 확인**: `http://localhost:3000` 접속 테스트
4. **ngrok 재시작**: `ngrok http 3000`

### 자동화된 방법
```cmd
# 서버 자동 시작
서버_시작.bat

# ngrok 설정 (새 CMD 창에서)
ngrok_문제해결_설정.bat
```

## 📋 체크리스트

### 서버 관련
- ✅ 서버가 실행 중인가?
- ✅ 포트 3000에서 응답하는가?
- ✅ 로컬에서 접속되는가?

### ngrok 관련
- ✅ ngrok 터널이 실행 중인가?
- ✅ 올바른 포트(3000)로 연결되어 있는가?
- ✅ 인증이 완료되었는가?

## 🔧 문제별 해결책

### 서버가 시작되지 않는 경우
1. **포트 충돌 확인**:
   ```cmd
   netstat -an | findstr ":3000"
   ```

2. **프로세스 종료**:
   ```cmd
   taskkill /f /im node.exe
   ```

3. **서버 재시작**:
   ```cmd
   npm run dev
   ```

### ngrok 연결 실패
1. **서버 상태 확인**
2. **ngrok 터널 재시작**
3. **방화벽 설정 확인**
