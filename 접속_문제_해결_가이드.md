# 🚨 SHE 디지털트윈 접속 문제 해결 가이드

## 🔍 문제 진단

### 현재 상황
- **호스트 IP**: `172.20.10.3`
- **접속 주소**: `http://172.20.10.3:3000`
- **문제**: 다른 사람이 접속할 수 없음

## 🛠️ 해결 방법

### 1단계: 서버 실행 확인

#### 서버가 실행 중인지 확인
1. **새로운 CMD 창 열기** (관리자 권한으로)
2. **프로젝트 폴더로 이동**:
   ```cmd
   cd "C:\Users\com\Desktop\SHE 디지털트윈"
   ```
3. **백엔드 서버 시작**:
   ```cmd
   cd backend
   npm start
   ```
4. **새로운 CMD 창에서 프론트엔드 시작**:
   ```cmd
   npm start
   ```

#### 서버 실행 확인 방법
- 브라우저에서 `http://localhost:3000` 접속 테스트
- 성공하면 서버가 정상 실행 중

### 2단계: 방화벽 설정

#### 방법 1: Windows 방화벽 GUI 설정
1. **Windows 설정** → **네트워크 및 인터넷** → **Windows 방화벽**
2. **고급 설정** 클릭
3. **인바운드 규칙** → **새 규칙**
4. **포트** 선택 → **다음**
5. **TCP** 선택, **특정 로컬 포트**에 `3000, 3001` 입력
6. **연결 허용** 선택
7. **모든 프로필** 선택
8. 이름: `SHE-Training-Site`

#### 방법 2: 관리자 권한 CMD
```cmd
netsh advfirewall firewall add rule name="SHE-Frontend-3000" dir=in action=allow protocol=TCP localport=3000
netsh advfirewall firewall add rule name="SHE-Backend-3001" dir=in action=allow protocol=TCP localport=3001
```

### 3단계: 네트워크 설정 확인

#### 같은 네트워크 확인
1. **호스트 컴퓨터와 접속하려는 컴퓨터가 같은 Wi-Fi에 연결되어 있는지 확인**
2. **IP 주소 범위 확인**:
   - 호스트: `172.20.10.3`
   - 접속자도 `172.20.10.x` 범위여야 함

#### 네트워크 연결 테스트
접속하려는 컴퓨터에서:
```cmd
ping 172.20.10.3
```

### 4단계: 브라우저 설정

#### 브라우저 캐시 클리어
1. **Ctrl + Shift + Delete**
2. **캐시 및 쿠키 삭제**

#### 다른 브라우저로 테스트
- Chrome, Firefox, Edge 등으로 테스트

## 🔧 추가 해결 방법

### 방법 1: 포트 변경
만약 포트가 차단되어 있다면:

1. **backend/src/config/index.js** 수정:
   ```javascript
   server: {
     port: process.env.PORT || 8080,  // 3001 → 8080
     host: process.env.HOST || '0.0.0.0'
   }
   ```

2. **package.json** 수정:
   ```json
   "scripts": {
     "start": "vite --port 8080 --host 0.0.0.0"
   }
   ```

3. **CORS 설정 업데이트**:
   ```javascript
   cors: {
     origin: [
       'http://localhost:8080',
       'http://172.20.10.3:8080'
     ]
   }
   ```

### 방법 2: 터널링 서비스 사용
- **ngrok** 사용 (무료)
- **localtunnel** 사용

### 방법 3: 클라우드 배포
- **Vercel** (프론트엔드)
- **Railway** (백엔드)
- **Heroku** (전체)

## 📱 모바일 접속 문제

### 모바일에서 접속이 안 될 때
1. **모바일 데이터 대신 Wi-Fi 사용**
2. **모바일 브라우저 캐시 클리어**
3. **다른 모바일 브라우저로 테스트**

## 🆘 최종 확인 사항

### 호스트 컴퓨터에서 확인
1. ✅ 서버가 실행 중인가?
2. ✅ `http://localhost:3000`에서 정상 작동하는가?
3. ✅ 방화벽이 포트를 차단하지 않는가?

### 접속하려는 컴퓨터에서 확인
1. ✅ 같은 Wi-Fi 네트워크에 연결되어 있는가?
2. ✅ `ping 172.20.10.3`이 성공하는가?
3. ✅ 브라우저 캐시를 클리어했는가?

## 📞 문제가 계속되면

1. **네트워크 관리자에게 문의**
2. **회사 방화벽 정책 확인**
3. **다른 네트워크에서 테스트**

---

**💡 팁**: 가장 간단한 해결책은 호스트 컴퓨터에서 서버를 다시 시작하고, 접속하려는 컴퓨터에서 브라우저 캐시를 클리어하는 것입니다.
