# 🌐 백엔드 연결이란? (초보자용 설명서)

## 📚 **백엔드 연결이 무엇인가요?**

### **간단한 비유로 설명하면...**

🏪 **식당에 비유해보세요:**
- **프론트엔드** = 식당의 매장 (고객이 보는 부분)
- **백엔드** = 식당의 주방 (고객이 보지 못하는 부분)
- **데이터베이스** = 식당의 창고 (재료 저장소)

고객(사용자)은 매장에서만 주문하고 음식을 받지만, 실제 요리는 주방에서 이루어집니다!

---

## 🔧 **SHE 디지털트윈에서 백엔드가 하는 일**

### **현재 상황 (프론트엔드만 사용)**
```
사용자 → 웹사이트 → 브라우저 메모리에 데이터 저장
```
- ✅ **장점**: 빠르고 간단함
- ❌ **단점**: 브라우저를 닫으면 데이터가 사라짐

### **백엔드 연결 시**
```
사용자 → 웹사이트 → 서버 → 데이터베이스에 영구 저장
```
- ✅ **장점**: 데이터가 영구 보존됨
- ✅ **장점**: 여러 사용자가 동시에 사용 가능
- ✅ **장점**: 실제 회사 시스템과 연동 가능

---

## 🎯 **백엔드 연결의 실제 활용**

### **1. 훈련 기록 영구 저장**
- 현재: 브라우저를 닫으면 훈련 기록이 사라짐
- 백엔드 연결 시: 모든 훈련 기록이 서버에 저장되어 영구 보존

### **2. 여러 사용자 동시 훈련**
- 현재: 한 번에 한 명만 훈련 가능
- 백엔드 연결 시: 여러 직원이 동시에 다른 시나리오 훈련 가능

### **3. 회사 조직도 연동**
- 현재: 가상의 조직도 사용
- 백엔드 연결 시: 실제 회사 조직도와 연동하여 현실적인 훈련

### **4. 관리자 기능**
- 현재: 개인별 훈련만 가능
- 백엔드 연결 시: 관리자가 전체 직원의 훈련 현황을 모니터링 가능

---

## 🚀 **백엔드 연결 설정 방법**

### **현재 상태 확인**
1. 웹사이트 우측 상단에 "백엔드 연결" 버튼이 있음
2. 이 버튼을 클릭하면 로그인 화면이 나타남

### **백엔드 연결 시 필요한 것**
1. **서버 주소**: 백엔드 서버가 실행되는 주소
2. **사용자 계정**: 서버에 등록된 사용자명과 비밀번호
3. **인터넷 연결**: 서버와 통신하기 위한 네트워크

---

## ⚙️ **기술적 세부사항 (고급 사용자용)**

### **프론트엔드 (Frontend)**
- **기술**: React, JavaScript, HTML, CSS
- **역할**: 사용자 인터페이스 제공
- **위치**: 사용자의 브라우저에서 실행

### **백엔드 (Backend)**
- **기술**: Node.js, Express, MongoDB/MySQL
- **역할**: 데이터 처리, 비즈니스 로직, API 제공
- **위치**: 서버 컴퓨터에서 실행

### **데이터베이스 (Database)**
- **역할**: 데이터 영구 저장
- **종류**: MongoDB (문서형), MySQL (관계형)

---

## 🔄 **데이터 흐름 비교**

### **프론트엔드만 사용할 때**
```
사용자 입력 → 브라우저 메모리 → 화면 표시
                ↓
         브라우저 종료 시 데이터 소실
```

### **백엔드 연결 시**
```
사용자 입력 → 프론트엔드 → 백엔드 API → 데이터베이스
                ↓
         화면 표시 ← 프론트엔드 ← 백엔드 응답 ← 데이터베이스
```

## ❓ **자주 묻는 질문**

### **Q: 백엔드 연결 없이도 훈련이 가능한가요?**
A: 네, 가능합니다! 현재 상태로도 모든 훈련 기능을 사용할 수 있습니다.

### **Q: 백엔드 연결 시 추가 비용이 있나요?**
A: 서버 운영 비용이 필요할 수 있지만, 소규모 사용 시에는 무료로 가능합니다.

### **Q: 데이터는 어디에 저장되나요?**
A: 프론트엔드만 사용 시 브라우저에, 백엔드 연결 시 서버의 데이터베이스에 저장됩니다.

### **Q: 보안은 어떻게 되나요?**
A: 백엔드 연결 시 사용자 인증, 데이터 암호화 등 보안 기능을 사용할 수 있습니다.
